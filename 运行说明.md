# 琼岛智划运行说明

本文档用于参赛演示和本地运行，写给不熟悉编程环境的使用者。

## 1. 打开 VS Code

打开 Visual Studio Code。如果电脑还没有安装 VS Code 或 Python，请先安装 VS Code 和 Python 3.10 以上版本。

## 2. 打开项目文件夹

在 VS Code 顶部菜单选择：

```text
文件 -> 打开文件夹
```

然后选择项目目录：

```text
D:\琼岛智划_project
```

## 3. 打开终端

在 VS Code 顶部菜单选择：

```text
终端 -> 新建终端
```

建议使用 PowerShell。

## 4. 安装依赖

在终端中输入：

```powershell
pip install -r requirements.txt
```

如果提示找不到 `pip` 或 `python`，请先确认 Python 已正确安装并加入系统 PATH。

## 5. 设置 API Key

如果需要使用 DeepSeek 在线解析功能，请在 PowerShell 中输入：

```powershell
$env:DEEPSEEK_API_KEY="你的APIKey"
$env:DEEPSEEK_BASE_URL="https://api.sydney-ai.com/v1"
$env:DEEPSEEK_MODEL="deepseek-chat"
```

请不要把真实 API Key 写入代码、文档或截图材料中。

## 6. 启动网页

在 PowerShell 中运行：

```powershell
python -m streamlit run app\streamlit_app.py --server.port 8502
```

启动成功后，终端会显示本地访问地址。

## 7. 浏览器打开 localhost:8502

在浏览器地址栏输入：

```text
http://localhost:8502
```

如果页面没有自动打开，可以手动复制上面的地址访问。

## 8. 输入需求测试

进入页面后，可以输入演示需求。例如：

```text
这个社区候鸟老人较多，希望医院、菜市场、公园都近一点，路上有休息座椅，过马路少。
```

也可以尝试年轻家庭、游客短租等场景，查看系统自动解析结果、规划图、动态 GeoJSON 图、空间统计表、多Agent分析和政策证据链。

## 9. 下载 Markdown / Word 报告

在页面的“一键生成规划报告”区域，可以预览报告并下载：

- Markdown 报告
- Word 报告

Word 报告依赖 `python-docx`。如果 Word 下载不可用，请重新安装依赖：

```powershell
pip install -r requirements.txt
```

## 10. 常见问题

### 为什么显示规则解析？

如果没有设置 API Key、网络不可用、DeepSeek 接口无响应，系统会自动使用规则解析兜底。规则解析用于保证演示系统在离线或接口异常时仍能运行。

### 为什么端口被占用？

如果 `8502` 端口已被其他程序占用，可以换一个端口启动，例如：

```powershell
python -m streamlit run app\streamlit_app.py --server.port 8503
```

然后在浏览器打开：

```text
http://localhost:8503
```

### 为什么 Word 下载不可用？

Word 下载依赖 `python-docx`。请先执行：

```powershell
pip install -r requirements.txt
```

如果仍不可用，可以先下载 Markdown 报告作为替代。

### 为什么找不到 GeoJSON？

动态 GeoJSON 图需要以下文件存在：

```text
D:\琼岛智划_project\data\dhm.geojson
D:\琼岛智划_project\data\hlg.geojson
```

如果文件缺失、为空或格式无效，页面会提示未找到有效 GeoJSON，并回退显示静态规划结果图。

### 为什么政策证据链为空？

政策证据链来自 `policy` 目录下的本地 txt 摘要文件。如果输入内容没有命中关键词、`policy` 文件缺失、文件为空或文件名不在项目目录下，系统会显示未检索到明确政策片段，并使用规则模板生成合规解释。

可以尝试输入包含以下关键词的需求，提高命中概率：

```text
医疗、养老、无障碍、休息座椅、公园绿地、学校、儿童、旅游、民宿、短租、免税、公交接驳、夜间出行、三亚、海南自由贸易港
```

## 手机端访问方式

1. 让手机和电脑连接同一个 Wi-Fi。
2. 在电脑端按前文步骤启动 Streamlit。
3. 在电脑终端查看本机局域网 IP。PowerShell 可输入：

```powershell
ipconfig
```

4. 找到当前 Wi-Fi 对应的 IPv4 地址，例如：

```text
192.168.1.100
```

5. 在手机浏览器访问：

```text
http://电脑IP:8502
```

例如：

```text
http://192.168.1.100:8502
```

如果手机无法访问，请检查：

- 手机和电脑是否在同一 Wi-Fi。
- Windows 防火墙是否拦截。
- Streamlit 是否正常运行。
- 端口是否为 `8502`。

## Streamlit Cloud 部署说明

如果需要通过公网链接给老师和同学访问，可以部署到 Streamlit Community Cloud。

1. 打开 Streamlit Community Cloud。
2. 使用 GitHub 账号登录。
3. 选择本项目所在 GitHub 仓库。
4. Branch 填写：

```text
main
```

5. Main file path 填写：

```text
app/streamlit_app.py
```

6. 在 Advanced settings / Secrets 中填写：

```toml
DEEPSEEK_API_KEY = "你的APIKey"
DEEPSEEK_BASE_URL = "https://api.sydney-ai.com/v1"
DEEPSEEK_MODEL = "deepseek-chat"
```

7. 点击 Deploy，等待 Streamlit Cloud 自动安装依赖并启动应用。

注意：不要把真实 API Key 写入 GitHub 仓库。Secrets 只填写在 Streamlit Cloud 后台配置中。若不配置 API Key，页面不会崩溃，会使用规则解析兜底。

## 11. 演示注意事项

- 当前系统采用“离线规划引擎 + 在线智能解释”架构。
- 展示端读取上游 PPO/SGNN 项目离线生成的 GeoJSON、PNG 和 CSV。
- 系统不进行现场 PPO/SGNN 训练。
- 系统不实时生成新的底层空间规划结果。
- DeepSeek、多Agent、RAG 主要用于需求解析、解释生成、合规辅助和报告生成。
- `policy` 目录为参赛演示用政策摘要知识库，不代表完整政策原文。
